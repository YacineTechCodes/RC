01 — Core Responsibilities

What the Network Layer
actually does

🏷️ Addressing

Assigns logical IP addresses to identify every machine on the network. Unlike MAC addresses (which are physical), IP addresses are hierarchical and routable across the entire internet.

🗺️ Routing

Determines the best path for data to travel from source to a distant destination. Routers examine each packet's destination IP and forward it hop by hop using routing tables.

🔁 Relaying

Forwards packets not destined for the local network toward their final destination. This is the core job of a router — receive, inspect, and relay.

🚦 Flow Control

Manages network congestion when too many packets circulate simultaneously. Prevents routers from being overwhelmed and maintains overall network stability.

Routing Strategies

Global vs. Local Routing
SCOPE
▼
Global strategies use knowledge of the entire network topology to make routing decisions — they can find the optimal path but require significant communication overhead.

Local strategies rely only on information about direct neighbors. Simpler and faster to update, but may not always find the globally optimal route.

Examples

OSPF (Global)RIP (Local)BGP (Global)
Static vs. Dynamic Routing
ADAPTATION
▼
Static routes are manually configured and fixed. They don't change unless an administrator updates them. Simple and predictable, but don't adapt to network failures.

Dynamic routes update automatically using routing protocols that share information about network changes, traffic load, and failures. Essential for large, complex networks like the Internet.

02 — IP Protocol & Datagrams

Inside every
IP packet

The Internet Protocol (IP) adds addressing information to every packet. Each packet carries a header containing metadata that routers use to forward it correctly. Hover over each field to learn what it does.

IPv4 Datagram Header — hover each field
Version
Protocol version. IPv4 = 4, IPv6 = 6.
IHL
Internet Header Length — size of the header in 32-bit words.
ToS
Type of Service — quality of service priority for this packet.
Total Length
Combined size of header + data in bytes. Max 65,535 bytes.
Identification
Unique ID to reassemble fragmented packets at the destination.
Flags
Control fragmentation. DF=Don't Fragment, MF=More Fragments follow.
Frag Offset
Position of this fragment within the original unfragmented datagram.
TTL
Time To Live — decremented at each router. Packet destroyed when it reaches 0, preventing infinite loops.
Protocol
Upper-layer protocol: 6=TCP, 17=UDP, 1=ICMP.
Header Checksum
Verifies the integrity of the header. Recalculated at each hop (TTL changes).
Source IP Address (32 bits)
The sender's IP address — 32 bits (4 bytes), e.g. 192.168.1.10
Destination IP Address (32 bits)
The receiver's IP address — 32 bits (4 bytes), e.g. 8.8.8.8
Data / Payload
The actual content being transported — handed down from the Transport layer.
⏱️

Time To Live (TTL)

Every packet starts with a TTL value (e.g., 64 or 128). Each router decrements it by 1. When it hits 0, the packet is discarded and an ICMP "Time Exceeded" message is sent back to the sender. This prevents packets from looping forever in the network.

🔒

Header Checksum

Since the TTL changes at every hop, the checksum must be recalculated at every router. It protects only the header — not the payload. The Transport layer (TCP) handles payload integrity separately.

04 — Address Resolution Protocol

Bridging IP
to MAC addresses

Network hardware only understands physical (MAC) addresses. ARP is the protocol that translates IP addresses into the MAC addresses needed to actually deliver a frame on the local network.

ARP Request / Reply — click to animate
Machine A
192.168.1.10
MAC: AA:BB:CC:DD:EE:01
→ "Who has 192.168.1.20?"
← "I do! My MAC is FF:..."
Machine B
192.168.1.20
MAC: FF:EE:DD:CC:BB:02
ARP Cache: Once B replies, A stores the IP→MAC mapping in its ARP cache so future packets skip the broadcast. The cache entry expires after a timeout to handle IP reassignments.
📢

ARP Request (Broadcast)

Machine A broadcasts to all devices on the local network: "Who has IP 192.168.1.20? Tell me your MAC." Every machine receives it, but only the one with that IP replies.

📬

ARP Reply (Unicast)

Machine B responds directly to Machine A with its MAC address. Machine A stores this in its ARP cache — a local table mapping IP→MAC for fast future lookups.

05 — Network Layer Hardware

Devices that operate
at Layer 3

Router

OSI Layer 3 Device

Routes packets between independent networks using its internal routing table. Examines each packet's destination IP, looks up the best next hop, and forwards it. Every hop on the internet is a router.

Routing table IP forwarding NAT DHCP

Gateway

Heterogeneous Network Connector

Connects networks of different types — for example, a local LAN to the internet. Goes beyond routing to handle protocol translation, advanced packet processing, and can serve as a firewall.

Routing Firewall QoS Protocol translation
