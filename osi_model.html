01 — Problem

Why do we need
standardization?

⚡

Heterogeneous Devices

Computing equipment comes from different manufacturers using different software platforms and communication protocols. A MacBook and a Windows server speak completely different native languages.

❓

The Core Problem

How do you make heterogeneous equipment communicate when they don't speak the same language? Without a shared ruleset, every device would be an island.

✅

The Solution: OSI

ISO (International Standards Organization) defined a reference model called Open Systems Interconnection (OSI) — a universal blueprint for network communication.

📦

7 Layers, 1 Goal

The model organizes communication into 7 distinct layers, each with a specific job. Hardware and software cooperate across these layers to move data from one end to the other.

Application

Serves as an interface for applications to access network services. This layer directly manages user applications — it's the part you actually see and interact with. Think email clients, browsers, file transfer utilities, and database software.

Presentation
DATA
▼
Determines the intermediate format used for exchanging data between computers. Acts as a translator for the application layer — handling encoding, compression, and encryption so that data is presented in a format the receiving system understands.

Common Protocols & Formats

ASN.1 SSL/TLS JPEG MPEG
5
Session
DATA
▼
Allows two remote applications to create, use, and terminate a connection. It ensures security and controls the dialogue between communicating processes — deciding when and for how long transmission takes place.

Common Protocols

NetBIOS RPC Session protocols
4
Transport
SEGMENTS
▼
Ensures packets are received without errors, in the correct order, without loss or duplication. Breaks long messages into small packets and reassembles them at the destination. Responsible for end-to-end transmission.

Common Protocols

TCP UDP NetBEUI IPX/SPX
3
Network
PACKETS
▼
Responsible for the addressing of messages. Determines the path packets must take based on the state of the network — this is routing. Every router in the internet operates primarily at this layer.

Common Protocols

IP ICMP CLNP RIP OSPF
2
Data Link
FRAMES
▼
Responsible for the error-free transfer of bit frames between two computers through the physical layer. Handles MAC addressing so that devices on the same local network can find each other.

Common Protocols

Ethernet Wi-Fi PPP HDLC Token Ring
1
Physical
BITS
▼
Transmits and manages the flow of raw electrical/optical signals through a physical medium. Defines the electrical, optical, mechanical, and functional interfaces with the cable or other medium. This is the actual wire, fiber, or radio wave.

Examples

RS232 10BASE-T X21 USB Bluetooth
03 — Services & Protocols

How layers
talk to each other

🔼

Services Flow Upward

Each layer provides services to the layer above it while consuming the services of the layer below. Layer 3 (Network) uses Layer 2 (Data Link) to deliver packets, and offers routing to Layer 4 (Transport).

🔗

Service Interface

A service interface is the set of primitives (function calls) that allow the layer above to access the current layer's services. It's like an API between layers.

🤝

Protocols: Peer-to-Peer

A communication protocol is defined between two layers of the same level — the Transport layer on Machine A communicates with the Transport layer on Machine B using the same protocol (e.g., TCP).

📋

Rules of Communication

For two computers to communicate, they must adopt the same rules. These rules constitute a protocol. Without agreed-upon protocols, data exchange would be impossible.

Encapsulation (Sending)

As data travels down the sender's layers, each layer wraps it with its own header — containing control info like destination address and message number. The data gets progressively bigger.

📂

Decapsulation (Receiving)

At the receiver, data travels up the layers. Each layer strips off its corresponding header and forwards the rest upward. By the time it reaches Layer 7, only the original data remains.
